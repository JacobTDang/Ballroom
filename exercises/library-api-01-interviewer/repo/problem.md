# Design the Library API

Design the public HTTP API for a lending library — books, copies,
members, checkouts.

That is the whole prompt — just like a real interview. Deciding what
the resources are, what operations exist, and what the wire format
looks like is your job: ask the interviewer (tutor pane, M-2) your
clarifying questions, then write the API spec in `solution.md` with
real resource names, methods, and example requests.

You have 45 minutes. When you're done, submit with M-q to reveal the
grading rubric, then ask the interviewer for a graded assessment.
