# Error Surface for a Payments API

Design the error model for a payments API: the codes, the response
shape, and the retry contract.

That is the whole prompt — just like a real interview. Deciding what
clients need from a failure, which canonical codes exist, and what
never leaks is your job: ask the interviewer (tutor pane, M-2) your
clarifying questions, then write the design in `solution.md` with
concrete shapes and worked examples.

You have 45 minutes. When you're done, submit with M-q to reveal the
grading rubric, then ask the interviewer for a graded assessment.
